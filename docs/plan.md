## アーキテクチャ判断

### フレームワーク不使用の理由
- 単一ページのシンプルなアプリケーション
- ビルドプロセス不要で GitHub Pages に直接デプロイ可能
- 依存関係ゼロでメンテナンスコストが低い
- パフォーマンスが良い (バンドルサイズなし)

### CSS Grid 選択の理由
- タイムテーブルの2次元レイアウトに最適
- セッションの任意の開始位置・期間を `grid-row` で直接表現可能
- `position: sticky` との組み合わせでヘッダ固定が容易

### Cookie 選択の理由 (暫定)
- 初期実装としてシンプル
- サードパーティライブラリ不要
- 将来的に localStorage に移行予定
